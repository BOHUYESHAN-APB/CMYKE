# VLA 模型选型结论（VRM Sandbox）

## 当前事实
- 本地加载路径：`local_models/modelscope/XiaomiRobotics/Xiaomi-Robotics-0-LIBERO`
- 这不是 base 预训练模型，而是 **LIBERO 场景微调版**。

## 是否现在就换模型？
**结论：暂不建议立刻切换到其他家模型。**

原因：
1. 当前主要瓶颈在控制层与环境映射（导航/交互触发/相机语义），不是纯模型容量。
2. Xiaomi 官方已给出该家族的任务化微调权重（LIBERO/CALVIN/SimplerEnv），你现在用的是对口的 LIBERO 版本。
3. 直接换模型会引入新的动作语义映射成本，短期内不一定优于继续优化策略层。

## 建议策略（泛化优先）
1. **先做控制层泛化**（强建议）
   - 统一目标接近策略（不要按对象类型写死规则）
   - 统一交互窗口判定（距离/朝向/遮挡）
   - 分层状态机：远距导航 -> 近距对齐 -> 交互

2. **再做同家族横向对比**（低风险）
   - 对比对象：
     - A: Xiaomi-Robotics-0-LIBERO（当前）
     - B: Xiaomi-Robotics-0（base，仅用于对照）
   - 核心指标：
     - 成功率（开门任务）
     - 平均步数
     - `INTERACT: too far` 次数
     - `unstuck` 触发次数
     - 推理耗时（P50/P95）

3. **达到门槛再考虑跨模型替换**
   - 若 A 在控制层稳定后仍低于目标：
     - 成功率 < 70%
     - 或 `unstuck` 频率 > 0.08 次/秒
   - 才进入跨模型候选验证（否则优先保持稳定栈）

## 一周执行计划（建议）
- Day 1-2: 完成分层状态机与日志指标采集
- Day 3: 跑 100 次门任务基准（固定指令集）
- Day 4: 引入 base 做 A/B 对照
- Day 5: 决策是否换模

## 决策门槛
- 若 LIBERO 版在同算力下：
  - 成功率提升 >= 10%
  - 且推理延迟不劣于 base 超过 15%
  - 则继续用 LIBERO 版
- 否则再评估替换路线
