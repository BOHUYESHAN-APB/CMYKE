# Windows 语音频道监听（虚拟声卡方案）

目标：把 Discord/KOOK 等语音频道的音频，通过虚拟声卡导入 CMYKE，再由系统 STT 转写成文本，并以“语音频道输入”注入对话（用于让 AI 能对语音频道内容做出回应）。

扩展目标（后续）：把 CMYKE 的 TTS 音频输出到虚拟声卡的“播放端”，让语音软件把它当成“麦克风输入”发到语音频道（语音回注入）。

## 前置

- Windows 10/11
- 一个虚拟声卡驱动
  - 典型选择：VB-Audio Virtual Cable（VB-CABLE）

## 推荐接线方式（Discord 示例）

1. 安装虚拟声卡驱动（例如 VB-CABLE）。
2. Discord 设置：
   - `语音和视频`：
      - `输出设备(播放)`：选择 `CABLE Input (VB-Audio Virtual Cable)`（让语音频道声音“打到虚拟声卡播放端/虚拟扬声器”）
      - 你自己的耳机听声音：
        - 建议用系统的“监听此设备”或额外的虚拟混音方案；不想折腾可以先只做转写验证
3. Windows 声音设置（二选一）：
    - 把 `CABLE Output (VB-Audio Virtual Cable)` 设为系统默认录音设备（系统 STT 将使用默认录音设备）
4. CMYKE：
    - `模型与能力配置` 中打开 `启用语音频道监听（Windows）`
    - 在聊天输入栏点“耳机”按钮开始监听

## 现状与边界

- 当前实现基于系统 STT：语音识别输入会**跟随 Windows 的默认录音设备**。
  - `模型与能力配置 -> 语音频道（Windows）` 中的设备下拉列表用于辅助你识别设备名称并跳转系统设置完成切换。
- Android 端默认不开放语音频道监听入口。
- 语音频道转写注入聊天时使用 `source=voiceChannel` 元数据，并在消息气泡上显示“语音频道”标签。

## 术语与接线（推荐记法）

- **语音软件输出(播放)** -> 虚拟声卡 **播放端（虚拟扬声器）**
- 虚拟声卡 **录音端（虚拟麦克风）** -> **Windows 默认录音设备** -> CMYKE (STT)
- （后续）CMYKE (TTS) -> 虚拟声卡 **播放端（虚拟扬声器）** -> 虚拟声卡 **录音端（虚拟麦克风）** -> 语音软件输入(麦克风)

提示：
- CMYKE 不会自动替你切换 Discord/KOOK 设备，也不会自动切换 Windows 默认设备；这些需要你在系统/语音软件设置里手动完成。
